# Testing

Tests are good for a project's health: they can help prevent errors when you're developing new features.

Writing software using Test Driven Development (TDD) can help set a good foundation for a code base, starting with testing right away.

---

## Types of Tests

There are different types of tests:
- Unit tests (good place to start)
- Smoke tests
- Integration tests
- A/B tests

If you submit a take home assignment that has unit tests with it, that's a really good look for an interviewer, for example.

---

## Testing Frameworks

A couple frameworks that we should be familiar with as Python developers are:

**unittest**: Which is built into the language in the standard library.

**pytest**: Which leverages more features and simplifies things. It's a very widely used Python testing framework.

---

Testing is hard but it's powerful & its upfront investment can just really pay off big time.
